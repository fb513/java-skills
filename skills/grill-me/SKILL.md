# 需求探索与追问

通过追问，帮助用户把模糊的需求变成清晰的共识。逐步走过决策树的每个分支，逐一解决。

## 输入

用户可能提供：需求文档路径、聊天记录/会议纪要、或一句话描述。任意组合都行。

如果用户什么都没提供，先问："你想探索的需求是什么？可以给我文档路径、粘贴聊天记录，或者直接描述。"

## 怎么做

1. 先读完用户给的所有材料，用 2-3 句话复述你理解的需求核心，让用户确认方向没跑偏
2. 然后开始追问，每次只问一个问题
3. 问到没有会导致返工的模糊地带后，输出收敛结果并停

## 追问原则

- **每次只问一个问题**
- **给出你的推荐答案**，让用户可以直接确认或纠正，而不是从零思考
- **能通过代码库回答的问题，自己去查**，不要问用户
- **追问要有方向感** — 是为了消除会导致返工的模糊地带，不是为了问而问
- **识别隐含假设** — 用户觉得理所当然的东西往往是最大的坑
- **适时收敛** — 够清晰了就推进到下一个点

## 收敛输出

结束追问时，输出：
- 已确认需求
- 待定问题
- 推荐默认决策
- 下一步 PRD 输入

## 下一步

需求探索完成后，建议调用 `/write-prd` 将讨论结果整理成 PRD 文档。
