# 生成 / 更新 PRD 文档

从当前对话上下文中提炼需求共识，生成新的 PRD 或更新已有的需求文档。

## 输入

- 当前对话中已讨论清楚的需求内容（通常是 grill-me 之后的上下文）
- 如果对话中用户之前提供过需求文档（grill-me 的输入），以该文档为基础更新，而非新建
- 用户也可指定输出路径，默认 `docs/prd/{日期}-{主题}.md`

如果对话中需求信息不足以生成 PRD，直接告诉用户哪些点还不清楚，建议先用 `/grill-me` 补齐。

## 怎么做

1. 回顾对话上下文，提取所有已达成共识的需求点和关键决策
2. **判断是新建还是更新**：如果对话中用户提供过需求文档，读取该文档，将讨论结果合并进去，保留原文档的结构和已有内容；如果没有已有文档，按下方模板新建
3. 写入文档，让用户确认是否需要调整

## 文档结构

```markdown
# {需求标题}

## 背景

为什么要做这个事，解决什么问题。

## 目标

这次要达成什么，用 1-3 条说清楚。

## 范围

### 包含
- ...

### 不包含
- ...

## 用户与场景

谁在什么场景下使用，入口在哪。

## 功能描述

按功能点逐条描述，每个功能点包含：
- 做什么
- 交互流程或接口行为
- 边界条件和异常处理

## 数据需求（如有）

新增或变更的数据结构、状态流转。

## 非功能性需求（如有）

性能、安全、兼容性等约束。
```

## 原则

- **只写对话中确认过的内容**，不自行补充未讨论的需求
- **语言简洁直接**，不写套话，每句话都要有信息量
- **粒度适中** — 够开发理解意图和边界就行，不需要写到接口字段级别
