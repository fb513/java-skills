# 后端技术设计生成

根据 PRD 文档，生成完整的后端技术设计：主设计文档 + 垂直切片文件。

---

## 输入

用户提供 PRD 文档路径（如 `docs/prd/20260417-后端技术需求.md`）。如果用户未指定 PRD 路径，主动询问。

---

## 约束

1. **不生成代码** — DDL 建表语句除外
2. **不确定就问** — 遇到不清楚的业务逻辑，停下来向用户提问，确认后再继续

---

## 执行流程

### Phase 1：探索项目 + 分析 PRD

探索现有代码，理解项目现状。重点关注：
- 已有哪些表和实体可以复用（通过 Entity 类推断）
- 已有哪些本地数据可以直接使用（避免不必要的跨库/跨服务调用）
- 项目的命名惯例和包结构

通读 PRD，如有不确定的点，整理问题一次性向用户确认。

### Phase 2：生成主设计文档

输出路径：`docs/plans/YYYYMMDD-<topic>/00-main-design.md`

文档结构：

```markdown
# <功能名称> — 主设计文档

> 基于 PRD `<PRD路径>`，面向后端开发。

## 1. 核心实体
表格列出所有实体（表名、说明），区分新建表和复用现有表。

## 2. 字段梳理
每个新建实体一个子章节，包含：
- 字段表格（字段名、类型、说明）
- 枚举映射表（如有）
- 索引建议
- 业务约束（如有）

## 3. 建表 DDL
完整 CREATE TABLE 语句（主键自增、字段带 COMMENT、含索引）。

## 4. 新增 Java 包结构
按分层列出新增的类文件。

## 5. 切片总览
| # | 切片名称 | 依赖 | 涉及接口 |

## 6. 设计决策说明
列出关键的架构选择及其理由。
```

### Phase 3：生成切片文件

每个切片一个文件：`docs/plans/YYYYMMDD-<topic>/NN-slice-<name>.md`

编号从 `01` 开始，按依赖顺序排列。

切片文件结构：

```markdown
# Slice NN: <切片名称>

| 依赖 | 交付物 |
|------|--------|
| 无 / #01, #02 | 交付物描述 |

## 涉及接口

### POST /api/xxx/create

**认证方式**: JWT

**请求**:
JSON 示例 + 字段说明表格

**响应**:
JSON 示例 + 字段说明表格

**实现逻辑**:
按步骤描述业务流程。

**错误场景**:
表格：场景、code、说明

## 验收标准

- [ ] ...
- [ ] 编译通过，无新增警告
```

### Phase 4：用户确认

展示产出，确认设计决策和切片粒度，迭代修改直到用户认可。
